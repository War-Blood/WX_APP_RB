# 编码规范

## 一、前端开发规程

### 1.1 技术栈规范
- **开发语言**: JavaScript (ES6+)
- **框架**: 微信小程序原生框架
- **运行环境**: 微信客户端基础库 2.20.0+

### 1.2 编码标准
- 使用 `const`/`let`，禁用 `var`
- 使用箭头函数、模板字符串
- 使用 `async/await` 处理异步
- 单文件不超过 300 行
- 文件头注释说明模块用途

### 1.3 文件组织结构
```
├── pages/              # 页面
│   ├── employee/       # 员工模块
│   └── admin/          # 管理员模块
├── components/         # 公共组件
├── services/           # 业务服务
└── utils/              # 工具函数
```

### 1.4 命名约定
| 类型 | 规范 | 示例 |
|-----|------|------|
| 变量/函数 | 小驼峰 | `getUserInfo` |
| 常量 | 全大写下划线 | `BASE_URL` |
| 组件 | 大驼峰 | `StatusTag` |
| 私有方法 | 下划线前缀 | `_loadData` |
| 文件名 | 小写连字符 | `report-edit.js` |

### 1.5 开发流程
- 页面生命周期：`onLoad` → `onShow` → `onReady`
- 数据绑定：`this.setData({})`
- 事件绑定：`bindtap` / `catchtap`
- 组件通信：`triggerEvent` 向上，`selectComponent` 向下

### 1.6 质量保障
- 移除 `console.log` 调试代码
- 用户输入必须校验和转义
- 测试核心功能流程

## 二、后端开发规程

### 2.1 技术栈规范
- **开发语言**: Node.js 18.x+
- **框架**: Express / 微信云函数
- **数据库**: MySQL 8.0
- **进程管理**: PM2

### 2.2 编码标准
- 使用 `const`/`let`，禁用 `var`
- 使用箭头函数、模板字符串
- 使用 `async/await` 处理异步
- 公共方法必须添加 JSDoc 注释

### 2.3 文件组织结构
```
├── cloudfunctions/     # 云函数
│   ├── submitReport/
│   ├── reviewReport/
│   └── manageAdmin/
└── sql/                # 数据库脚本
```

### 2.4 命名约定
| 类型 | 规范 | 示例 |
|-----|------|------|
| 变量/函数 | 小驼峰 | `getPool` |
| 常量 | 全大写下划线 | `MYSQL_HOST` |
| 数据库表 | 小写下划线 | `daily_reports` |
| API路径 | 小写连字符 | `/api/report-list` |

### 2.5 开发流程
- 统一返回格式：`{ code: 0, data: {}, message: '' }`
- 错误码：`0` 成功，`-1` 通用错误，`403` 无权限，`404` 未找到
- 使用连接池管理数据库连接

### 2.6 质量保障
- 敏感信息使用环境变量，禁止硬编码
- SQL 使用参数化查询，防止注入
- API 返回数据过滤敏感字段
- 错误日志记录完整堆栈信息

## 三、通用规范

### 3.1 Git提交规范
| 类型 | 说明 | 示例 |
|-----|------|------|
| `feat` | 新功能 | `feat: 添加日报导出` |
| `fix` | 修复Bug | `fix: 修复登录问题` |
| `refactor` | 重构 | `refactor: 优化请求` |
| `docs` | 文档 | `docs: 更新README` |

### 3.2 安全规范
- 敏感信息使用环境变量
- SQL 使用参数化查询
- 用户输入必须校验和转义
